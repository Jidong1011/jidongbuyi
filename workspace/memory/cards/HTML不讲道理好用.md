# HTML的不讲道理好用 速查卡

> **一句话本质**：Anthropic工程师Thariq说，AI写的Markdown计划太长人不读了，换成HTML（有卡片、颜色、按钮）人才愿意停下来做决策

## 核心三要素

1. **Markdown太长没人读**：AI能写上万字，人类逐行阅读的耐心已经没了
2. **HTML是人机协作界面**：卡片、表格、mockup让人类"扫读→选择→参与"
3. **Compute Allocator**：AI跑8小时=500美元，你必须在开始前想清楚让它做什么

## 一句话类比

Markdown像一本纯文字菜单，HTML像餐厅门口的展示窗——你不需要读完菜单才知道点什么，看一眼展示窗就能做决定。

失效边界：线性阅读内容（小说/论文）用Markdown更好；HTML优势在于"需要人做决策"的场景。
