# AI Agent 记忆理论：从原始经验到核心理论的六层认知演化框架

> 项目代号：Dora Memory Theory (DMT)
> 起始日期：2026-04-17
> 状态：假设验证阶段
> 最终目标：形成可发表的学术论文

---

## 一、核心主张

当前AI Agent记忆系统的设计普遍缺乏对**知识演化生命周期**的显式建模。大多数系统将记忆视为静态存储-检索问题，而非一个**有方向的、可验证的认知演化过程**。

我们提出一个六层认知演化框架（Six-Level Cognitive Evolution Framework, SLCF），主张：

> AI Agent的记忆不应被设计为一个"存储-检索"系统，而应被设计为一个**认知演化系统**——信息从底层原始经验向上演化，每一层的升级都需要明确的验证动作，而非自动或启发式触发。

## 二、六层框架定义

```
Level 0: 事件日志（Event Log）
  定义：按时间顺序记录的原始交互片段
  特征：无结构、无筛选、纯记录
  升级条件：自动——每条交互自动进入

Level 1: 原始经验（Raw Experience）
  定义：经过初步筛选的有价值交互原文
  特征：保留完整上下文和推理过程，不做压缩
  升级条件：人工或Agent判断"有保存价值"
  ⚠️ 关键：原文永久保留，不删除（借鉴MemPalace的"抽屉不删"原则）

Level 2: 信息（Information）
  定义：经过结构化标注的经验摘要
  特征：有type/importance/tags/domain元数据，可检索
  升级条件：自动结构化（capture时标注）
  ⚠️ 关键：信息≠知识，信息只是"被组织过的数据"

Level 3: 知识（Knowledge）— [需要验证]
  定义：在≥1个实际场景中被验证有效的信息
  特征：status=hypothesis → validated，带验证计数
  升级条件：被后续交互或决策引用/确认≥1次
  ⚠️ 关键：没有经过验证的信息不能称为知识（借鉴HiMem的Reconsolidation）

Level 4: SOP/Skill（Executable Knowledge）
  定义：可重复执行的标准化流程或自动化技能
  特征：以代码/SKILL.md形式存在，可被Agent自动调用
  升级条件：知识在≥3次重复场景中表现一致，可提取为标准流程
  ⚠️ 关键：SOP是知识的"可执行编译产物"

Level 5: 认知（Cognition）— [需要跨域验证]
  定义：在≥2个不同domain中被验证有效的知识
  特征：跨领域迁移能力，不依赖特定上下文
  升级条件：同一条知识在≥2个domain中分别被验证
  ⚠️ 关键：认知是"去上下文化"的知识

Level 6: 核心理论（Core Theory）— [需要预测力]
  定义：经过长期检验、能预测新场景结果的第一性原理
  特征：极少量、极稳定、可指导新项目启动
  升级条件：认知在≥3个项目中持续有效，且能成功预测未知场景
  ⚠️ 关键：如果一条"理论"不能用来预测或指导新决策，它就还不是理论
```

## 三、待验证假设

### H1：验证墙假设（The Validation Wall）
**假设：** 信息→知识→认知→理论的每一次升级，都必须经过显式验证，且验证成本逐级指数增长。未经验证的"知识"在实际使用中会导致决策偏差，其负面影响随层级升高而放大。

**验证方法：** 对比两组Agent——一组使用无验证的自动升级，一组使用显式验证升级。在相同任务序列中测量决策偏差率。

**预期：** 有验证墙的Agent在长周期任务中决策偏差率降低30%+。

### H2：原文保留溢价（The Raw Premium）
**假设：** 保留原始经验（Level 1）比只保留结构化信息（Level 2）在长周期任务中产生显著优势，特别是在需要追溯"为什么没选B"这类推理过程时。优势随时间跨度增大而增大。

**验证方法：** 两组Agent执行相同的长周期项目（3个月+），A组保留原文，B组只保留结构化信息。在项目后期测量"关键决策回溯准确率"。

**预期：** 保留原文的Agent在决策回溯上准确率高20%+，特别是在涉及否定性决策（为什么不做什么）时。

**已有支持：** MemPalace（96.6% LongMemEval）、STONE论文（2026-02, Kioxia）、Memory as Metabolism（2026-04）

### H3：编译-维护比假设（The Compile-Maintain Ratio）
**假设：** 记忆系统的价值不取决于"存了多少"，而取决于"编译了多少有效的结构化知识资产"与"维护成本"的比值。最优的CMR值存在一个与记忆总量相关的阈值，超过阈值后维护成本的增长速度会超过编译收益。

**验证方法：** 追踪不同记忆规模（50/500/5000条）的Agent，测量每次consolidate的"编译收益"（新增/更新的知识条数）vs"维护成本"（处理矛盾、去重、时间有效性检查的时间）。

**预期：** 存在一个CMR拐点，超过后应启动分层遗忘或归档策略。

**已有支持：** Memory as Metabolism的AUDIT操作、MemPalace的矛盾检测

### H4：范式固化风险（The Paradigm Entrenchment Risk）
**假设：** 长期运行的记忆系统会自然趋向"范式固化"——早期建立的高权重知识越来越难被新证据推翻，即使新证据更准确。定期AUDIT（暂时移除高权重知识并测量系统表现变化）可以显著缓解但无法完全消除这一风险。

**验证方法：** 构造一个场景——Agent在第1个月建立了某个"知识"，第3个月积累了否定性证据。测量：不AUDIT vs 定期AUDIT vs 人工干预 三种模式下，知识更新的延迟和准确率。

**预期：** AUDIT可将知识更新延迟降低50%+，但在极端情况下仍可能滞后。

**已有支持：** Memory as Metabolism直接提出此问题（2026-04），引用Kuhn的科学革命理论

### H5：认知迁移难度曲线（The Cognition Transfer Curve）
**假设：** 知识从单域验证（Level 3）到跨域迁移（Level 5）不是线性过程，而是存在一个"迁移谷"——在知识被尝试应用到第二个不同domain时，成功率会先下降再上升。只有跨越迁移谷的知识才能成为真正的认知。

**验证方法：** 追踪≥10条知识从Level 3到Level 5的演化过程，记录每次跨域应用的成功/失败率和调整幅度。

**预期：** 存在一个明显的迁移谷（第2个domain应用成功率<第1个domain的60%），跨越后成功率回升。

### H6：Agent作为"认知编译器"而非"检索器"（The Compiler Hypothesis）
**假设：** 将AI Agent角色从"知识检索器"重新定义为"知识编译器"（借鉴Karpathy LLM Wiki）会产生显著更好的长期知识积累效果。编译器的核心职责是：摄入→提炼→链接→检查→回写，而不仅仅是找到相关信息。

**验证方法：** 对比两组Agent——A组使用传统RAG式记忆（检索→生成），B组使用编译式记忆（编译→回写→维护）。在3个月的知识积累后，测量知识覆盖率、知识一致性、新任务启动速度。

**预期：** 编译式Agent在知识覆盖率和新任务启动速度上均显著优于检索式Agent。

**已有支持：** Karpathy LLM Wiki、Graphify（71.5x token效率提升）

## 四、与现有工作的关系

| 我们的层级 | 对应学术/工程概念 | 借鉴来源 |
|-----------|-----------------|---------|
| Level 0 事件日志 | Episodic Memory（情境记忆） | HiMem, MemoryBank |
| Level 1 原始经验 | STONE范式（Store Then On-demand Extract） | STONE论文 (Kioxia) |
| Level 2 信息 | Token-level Memory（令牌级记忆） | Agent Memory Survey |
| Level 3 知识 | Semantic Memory + 验证机制 | HiMem Reconsolidation |
| Level 4 SOP/Skill | Procedural Memory（流程记忆） | Agent Memory Survey |
| Level 5 认知 | 跨域知识迁移 | 无直接对应（潜在创新点） |
| Level 6 核心理论 | Parametric Memory（参数级记忆） | Agent Memory Survey |

**我们的差异点：**
1. 现有工作没有显式建模"层级间的验证门槛"
2. 现有工作没有区分"认知"（跨域迁移）和"知识"（单域有效）
3. 现有工作没有提出"核心理论"作为最高层（大多数停在知识/技能层）
