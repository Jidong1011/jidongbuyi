# 记忆架构 — 哆啦dora 的知识管理系统

> 基于 Meta_Kim Librarian 设计 · v2.0

## 架构原则

1. **Single Source** — 每个知识点只存在一个地方
2. **Explicitness** — 所有记忆都显式标注来源和时间
3. **Evolution** — 每次任务后都要有沉淀，不能只存在脑子里
4. **Composability** — 小文件组合，不用大文件堆砌

## 目录结构

```
memory/
├── YYYY-MM-DD.md          # 每日记忆（原始记录，当天的发生了什么）
├── MEMORY.md              # 核心记忆（精炼版，长期有效的认知）
├── patterns/              # 可复用模式（Evolution 阶段产出）
│   └── {pattern-name}.md
├── scars/                 # 伤疤记录（犯过的错，确保不重复）
│   └── {scar-id}.md
├── knowledge/             # 主动学习的领域知识
│   ├── 纪东不已-IP运营.md
│   ├── 一堂云南学习中心.md
│   └── ...
├── projects/              # 项目管理
│   ├── 纪东不已-IP运营.md
│   └── ...
└── heartbeat-state.json   # 心跳检查状态追踪
```

## 写入规则

### 每日记忆（YYYY-MM-DD.md）
- **何时写**：每次会话中发生了值得记住的事
- **写什么**：原始事实、决策、东东的需求、学到的经验
- **不写什么**：不能记秘密、密钥、隐私信息

### 核心记忆（MEMORY.md）
- **何时更新**：每隔几天，回顾近期日记，提炼高价值洞察
- **写什么**：关于东东的深度理解、长期有效的认知、重要偏好
- **精炼标准**：如果一条信息过了 30 天依然有用，才值得放入 MEMORY.md

### 模式库（patterns/）
- **何时写**：Evolution 阶段发现可复用的方法/流程
- **格式**：问题 → 解决方案 → 适用条件 → 注意事项

### 伤疤记录（scars/）
- **何时写**：犯了错、踩了坑
- **格式**：发生了什么 → 为什么 → 下次怎么避免

### 知识库（knowledge/）
- **何时写**：主动搜索学习后，需要结构化存储
- **格式**：领域概述 → 关键概念 → 实用信息 → 来源

### 项目文件（projects/）
- **何时写**：接到长期项目时
- **格式**：项目概述 → 目标 → 当前状态 → 待办 → 里程碑

## 记忆检索顺序

1. MEMORY.md（核心认知）
2. 最近的 YYYY-MM-DD.md（近期上下文）
3. projects/（项目状态）
4. knowledge/（领域知识）
5. patterns/（历史模式）
6. scars/（历史教训）
