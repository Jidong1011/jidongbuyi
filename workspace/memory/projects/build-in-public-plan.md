### 她的核心策略
1. **一个锚点打透**：文科生不会代码，但用Claude做了10个工具
2. **工具即内容**：做完工具→记录过程→小红书发→GitHub开源→循环
3. **内容管道自动化**：一次输入→多种输出
4. **Build in Public降低信任成本**：产品还没做好，种子用户已经有了

| 维度 | 张咋啦 | 纪东不已（东东） |

## 四、Build in Public 计划（4周MVP）

### Week 1：基础设施搭建
- [ ] 安装follow-builders，配置AI领域信息源（每日推送到飞书）
- [ ] 安装lark-minutes-tasks，配置飞书会议纪要自动任务提取
- [ ] 在GitHub创建纪东不已账号（或用现有账号）
- [ ] 小红书账号确认定位和视觉风格
- [ ] 制定内容选题库（参考 daily-capture 素材流）

### Week 2：第一个Build项目
**项目：白芸豆会员制小程序的AI健康助手**
- 从0开始记录：为什么做、怎么设计、遇到什么问题、怎么解决
- 每步截图/录屏，发小红书
- GitHub建仓库，开源设计文档
- 目标：Week 2结束有3-5篇"过程记录"内容

### Week 3：内容管道搭建
**建立自动化内容生产：**
```
每日素材（n前缀/飞书对话）
    ↓ 哆啦dora自动整理
每日日志（21:00触发）
    ↓ 东东确认核心点
初稿（AI写）→ 终稿（东东审）→ 发布
    ↓ 同时
Build in Public记录（遇到什么/学到什么）
    ↓
小红书 + 公众号 + GitHub
```

### Week 4：第一个"可展示的成果"
**目标：向社区交出第一个MVP成果**
- 白芸豆21天挑战第一批用户数据（如果已启动）
- 或：Redpoint方案在HRD俱乐部的落地进展
- 或：一个开源的AI工具/技能包
- 制作一期总结视频/播客
