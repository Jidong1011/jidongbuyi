## 一、战略定位

### 一段话定义

Redpoint 是一个带真实企业任务实践的成长服务。人才通过课程学习→领取真实企业任务→在实战中验证和强化能力→获得成长数据反馈，完成"学-练-战-证"闭环。人才为成长付费，企业为任务结果和精准人才数据付费。

### 商业模型

```
┌─────────────────────────────────────────────────┐
│                   Redpoint                       │
├─────────────┬──────────────┬────────────────────┤
│  成长者付费  │   实践闭环    │    企业付费        │
│  1500-2000元 │              │                    │
│             │  课程学习 ──→ │  CEO/业务负责人     │
│  红点定位课  │  领取任务 ──→ │  为任务结果付费     │
│  德鲁克5习惯 │  实战交付 ──→ │                    │
│  AI评估反馈  │  能力数据 ──→ │  HR               │
│             │              │  为人才推荐付费      │
├─────────────┴──────────────┴────────────────────┤
│           AI驱动（90%自动化）                     │
│  课程交付 · 评估打分 · 任务匹配 · 运营触达        │
└─────────────────────────────────────────────────┘
```

### 三个核心差异化

| 维度 | 传统知识付费 | 传统招聘 | Redpoint |
|------|------------|---------|----------|
| **实践闭环** | 学完即止，无真实场景 | 只看简历和面试，不看实际能力 | 真实企业任务是课程的必经实践环节 |
| **价值交付** | 靠内容质量和情绪价值 | 靠职位匹配成功率 | 靠成长可感知（前7天即时价值）+ 实战成果 |
| **数据沉淀** | 无个人成长数据 | 静态简历快照 | 动态能力档案：方向+画像+加速度+轨迹 |

### 战略目标时间线

- **12周（2026.07.10）：** 跑通MVP，验证付费意愿，企业端产生首笔收入
- **24周（2026.10.07）：** 单月营收过10万，复购率≥30%，50+活跃成长者
- **48周（2027.03.30）：** 月营收过30万，企业端收入占比≥30%，形成口碑飞轮

## 三、各Phase详细方案

### Phase 0：准备期（4/15-4/20）

**目标：** HRD俱乐部主席会面达成合作意向，拿到≥2个企业真实任务需求

**核心动作：**

1. **周末会面准备清单（4/17前完成）**
   - [ ] 准备1页Redpoint简介（聚焦HR价值：精准人才推荐+真实任务场景）
   - [ ] 准备3个示例任务模板（市场调研、竞品分析、用户访谈脚本）
   - [ ] 明确诉求：①HRD俱乐部成员企业提供任务 ②推荐种子用户 ③复用俱乐部社群触达
   - [ ] 准备合作提案：企业免费出任务→获得成果+人才数据→优先推荐权

2. **会面核心议题**
   - Redpoint对HRD俱乐部的价值（会员企业的免费项目外包+人才储备）
   - 合作方式（不做正式合作框架，先跑通再谈）
   - 下一期活动东东分享的主题和时间

3. **会后72小时follow-up（4/21前）**
   - 发送会议纪要+合作要点确认
   - 追问3家具体企业的任务需求对接人
   - 确认在俱乐部社群/活动中的曝光机会
   - 如果主席未明确支持→联系2位关系最近的HRD成员单独沟通

**关键产出：** ≥2个企业任务需求确认 + 种子用户招募渠道确认

### Phase 1：产品构建（4/21-5/4）

**目标：** 课程可交付，AI流程跑通，≥3个企业任务入库，种子用户招募启动

**1. 课程产品化**

- **红点定位课：** 标准化为3模块×2课时
  - 模块1：自我拆解（优势-价值观-经历映射）
  - 模块2：方向探索（行业×职能交叉分析）
  - 模块3：红点卡初版输出
- **德鲁克5习惯：** 每个习惯=1次课程+1次实践任务
  - 习惯1（重视贡献）→ 对应企业任务：价值主张梳理
  - 习惯2（聚焦优势）→ 对应企业任务：能力盘点报告
  - 习惯3（时间管理）→ 对应企业任务：日程优化方案
  - 习惯4（要事优先）→ 对应企业任务：项目优先级排序
  - 习惯5（有效决策）→ 对应企业任务：决策分析报告
- **交付形式：** AI直播/录播课（Coze工作流自动生成）+ 课后作业AI评估 + 每周1次东东直播答疑（30分钟）

**2. 获取3-5个企业真实任务**
- 从HRD俱乐部对接的任务中筛选标准化程度高的
- 每个任务必须包含：任务描述、交付物标准、评估维度、完成时限（7天内）
- 任务定价：免费（MVP阶段企业不出钱，换人才数据和成果）

**3. AI自动化搭建（Coze工作流）**
- 用户注册→红点定位测评→课程推送
- 作业提交→AI评估→反馈生成
- 任务匹配→进度追踪→结果收集
- 企业报告生成（人才数据集四维）

**4. 种子用户招募**
- 渠道：HRD俱乐部推荐 + 一堂社群 + 东东个人朋友圈
- 定价：种子用户999元（原价1500的早鸟价）
- 招募话术重点："真实企业项目经验+能力成长证明"

**关键产出：** 课程包上线 + AI工作流测试通过 + ≥3个任务入库 + 招募文案发布

### Phase 2：MVP验证（5/5-5/18）

**目标：** 10人付费报名，≥6人在7天内感知到即时价值，NPS≥7

**1. 7天首次价值交付设计（关键修正）**
- Day 1：红点定位测评 → 生成个人定位初版报告（即时获得感）
- Day 2-3：红点课模块1-2 → 输出"我的优势-方向矩阵"
- Day 4：领取第一个企业任务 → 明确知道要做什么
- Day 5-6：任务执行+AI辅导
- Day 7：提交任务成果 → AI评估+企业反馈 → 获得"实战成果证明"
- 前7天核心承诺："拿到你的第一份真实企业项目成果"

**2. 全链路验证**
- 报名→支付→入群→课程→作业→任务→评估→反馈
- 每个环节记录卡点时间和用户反馈

**3. 成功标准**
- 10人付费（≥8人完成Day 1，≥6人完成7天全流程）
- NPS问卷（Day 7发送）：≥7分
- ≥3人完成至少1个企业任务交付
- 收集≥10条可改进的具体反馈

**关键产出：** 10人MVP数据 + NPS报告 + 全链路卡点清单 + 改进方案

---

### Phase 3：数据积累（5/19-6/15）

**目标：** 50人参与（含免费试用课转化），人才数据集模型跑通，企业端首笔收入

**1. 扩大规模与转化漏斗**
- 免费试用课上线（红点定位测评+1节体验课），作为获客入口
- 漏斗：免费课体验 → 付费课转化（目标转化率15%→50人×15%=7-8人直接转化）
- 其他渠道：HRD俱乐部推荐、口碑推荐、一堂社群
- 50人目标：30人付费+20人免费试用课转化

**2. 人才数据集四维模型验证**
- 红点方向：从测评+任务选择倾向推导
- 能力画像：从作业评分+任务成果质量评估
- 成长加速度：Day 1 vs Day 7 vs Day 14的能力变化曲线
- 成长轨迹：时间线可视化记录
- **关键假设验证：** AI评估分数是否与人工评估一致（首批10人的AI评分 vs 东东人工评分对比）

**3. 企业端收入验证**
- 向3-5家合作企业发送"人才推荐报告"
- 定价模型测试：企业按推荐人数付费（500元/人）或按任务成果付费（1000元/任务）
- 目标：≥1家企业产生付费行为

**4. AI自动化优化**
- 根据Phase 2反馈优化AI评估准确度
- 自动化覆盖率从60%→85%

**关键产出：** 人才数据集v1.0 + AI评估准确度报告 + 企业端首笔收入凭证

---

### Phase 4：飞轮启动（6/16-7/10）

**目标：** 第二期课程复购率≥25%，企业持续发单≥5个/月，验证单位经济模型

**1. 复购率验证**
- 第一期用户完成课程后推送第二期（进阶课或新任务）
- 目标：≥8人（50人中的25%）报名第二期
- 复购定价：1500元（恢复原价）

**2. 企业端持续运营**
- 企业任务发布流程标准化（需求模板+定价标准）
- 目标：≥5个企业任务/月（新增企业≥2家）
- 企业满意度调研

**3. 口碑传播设计**
- 完课用户生成"成长报告"（可分享到朋友圈）
- 推荐1人报名→双方各获200元优惠
- 目标：≥20%的新用户来自推荐

**4. 单位经济模型验证**
- CAC（获客成本）：目标≤300元/人
- LTV（用户生命周期价值）：首期1500×复购率25%=1875元
- 毛利率：AI成本极低，目标≥70%
- 企业端ARPU：目标≥500元/企业/月

**关键产出：** 单位经济模型报告 + 飞轮运转证据 + 12周复盘文档

## 五、关键指标体系

### 北极星指标
**每周完成任务交付的成长者人数**（反映核心闭环运转效率）

### 分层指标

**人才端：**
- 报名转化率（免费→付费）
- 7天完成率
- 课程完课率
- NPS（Day 7 / Day 14 / 课程结束）
- 任务交付率
- 复购率

**企业端：**
- 活跃企业数
- 月发布任务数
- 任务完成率（企业对成果满意度）
- 企业付费转化率
- 企业续约率

**财务：**
- 月营收（人才端+企业端）
- CAC
- LTV
- 毛利率
- 企业端收入占比

**运营/AI：**
- AI自动化覆盖率
- AI评估与人工评分偏差
- 平均响应时间
- 东东周均投入时间

## 六、风险预案

| # | 风险 | 触发条件 | 应对方案 |
|---|------|---------|---------|
| 1 | **没人愿意付费** | M3报名<6人 | ①推出免费红点定位测评（10分钟出结果）做引流 ②定价降至499元（限时） ③如果仍<3人→暂停，重新验证需求（访谈20个目标用户） |
| 2 | **7天价值交付失败** | M4完成率<40%或NPS<5 | ①立即电话访谈所有未完成用户 ②如果是课程问题→压缩为3天核心交付 ③如果是任务太难→东东做1个完整示范 ④如果是AI体验差→东东接手评估，AI做辅助 |
| 3 | **企业不愿出任务** | M1后2周内0个企业任务 | ①东东自己设计3个"通用型"企业任务（不依赖具体企业） ②联系一堂的创业学员企业 ③从公开商业案例中提取模拟任务（标注"模拟"，降级但不停 |
| 4 | **AI评估不可信** | Phase 2中AI评分与人工偏差>30% | ①Phase 3前10人全部人工复核 ②标记AI弱点维度（如创意类）人工接管 ③如果整体偏差>40%→取消AI评分，改为自评+互评+东东抽检 |
| 5 | **东东时间爆炸** | Phase 2后东东周投入>20小时 | ①立刻砍掉东东手动做的事（答疑改录播、评估全交AI） ②招募1个兼职助教（从第一期优秀学员中选） ③如果仍超时→缩小规模，不追人数追完成率 |
