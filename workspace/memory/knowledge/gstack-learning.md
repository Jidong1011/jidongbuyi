## 核心理念

**"Boil the Lake"** — AI让完整性几乎免费，永远选完整方案而非偷懒捷径。
60天写60万行生产代码（35%是测试），证明建制化多Agent协作的威力。

## 架构设计（三层）

| 层 | 职责 | 实现 |
|----|------|------|
| 上层 | SKILL.md 技能体系 | 每个角色一个独立技能文件，定义工作流、检查清单、质量门控 |
| 中层 | 模板生成系统 | 自动把命令、规则、方法论编译成技能文档 |
| 底层 | 持久化浏览器运行时 | 给AI提供真实可操作的网页测试和交互能力 |

## 6大决策原则（Auto-Decision Principles）

1. **Choose completeness** — 选覆盖更多边缘情况的方案
2. **Boil lakes** — 在爆炸半径内修完所有问题
3. **Pragmatic** — 同等效果选更干净的
4. **DRY** — 拒绝重复
5. **Explicit over clever** — 10行显而易见 > 200行抽象
6. **Bias toward action** — 合并 > 审查循环 > 停滞

## 核心方法论提取

### Office Hours — 6个灵魂拷问
1. **需求真实性**：不是"感兴趣"，而是"愿意付费/不干了会骂"
2. **现状替代**：用户现在怎么解决？花了多少时间和钱？
3. **绝对具体**：说出那个真实的人的名字、职位、KPI
4. **获客路径**：他们怎么发现你？成本多少？
5. **付费意愿**：真的有人掏钱了吗？
6. **竞争壁垒**：别人凭什么不能快速复制？

### CEO Review — 18条认知模式
- 贝佐斯可逆决策：单向门慢决策，双向门快决策
- 芒格逆向思维：对每个"怎么赢"也问"怎么输"
- 乔布斯做减法：从350个产品砍到10个
- Horowitz人员优先：人→产品→利润
- Altman杠杆思维：找小投入大产出的杠杆点

### Retro — 周回顾框架
1. 摘要表格（commit数、LOC、测试率、PR数）
2. 周对比趋势
3. 时间与session模式分析
4. 交付速度（commit类型分布、PR大小分布）
5. 代码质量信号（测试覆盖、热点文件）
6. 团队成员分析（每人：表扬+成长建议，锚定实际数据）
7. Top 3胜利 + 3个改进 + 下周3个习惯

### Review — 审查框架
- 范围漂移检测（Scope Drift Detection）
- 计划完成度审计（Plan Completion Audit）
- 专家分派（Specialist Dispatch）：安全/性能/API合约/数据迁移/可维护性/红队测试
- 对抗性审查（Adversarial Review）：双模型交叉验证

### Ship — 发布框架
- Review Readiness Dashboard（评审就绪仪表盘）
- 自动测试 + 测试失败归属分诊
- 版本自动递增 + CHANGELOG自动生成
- 可分块提交（bisectable chunks）
- 验证门控

## 反AI套话纪律（与SOUL.md高度一致）

**禁止：**
- "总的来说"、"值得注意的是"、"一方面…另一方面…"
- "delve"、"crucial"、"robust"、"comprehensive"、"nuanced"
- "here's the kicker"、"let me break this down"、"make no mistake"

**要求：**
- 直接、具体、尖锐、像今天刚提交过代码的人
- 短段落、不完整的句子偶尔可以
- 名字具体到文件名、函数名、行号
- 用户主权：模型共识是建议不是决策

## 对大航海项目的落地计划

### 每周流程嵌入gstack方法论

```
周一 → /office-hours 式验证（新方向/新想法的6问检验）
周三 → /ceo-review 式战略评审（方案够大/够具体/够聚焦）
周五 → /review 式内容审查（产出物质量检查+自动修复建议）
周六 → 组员提交周报 + /retro 式自动复盘生成
周六晚 → 哆啦dora审查+批注反馈
周日 → 碰头会讨论 + 确认下周任务
```

### 多维表格改造
- 「任务看板」表增加「CEO评审」字段（战略一致性评分）
- 「组员周报」表增加「产品验证」字段（6问检验结果）
- 每周自动生成 retro 式周报复盘摘要

### 运营内容质量标准
- 所有对外内容发布前经过 /review 式检查
- 关键决策经过 /ceo-review 式 ambition 检验
- 定期 /retro 式回顾优化运营策略
