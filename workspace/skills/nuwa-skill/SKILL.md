# 女娲 · 认知蒸馏引擎

> 输入一个名字，还你一套可运行的认知操作系统。

## 触发词
"蒸馏 XXX" / "造一个 XXX 的 Skill" / "distill XXX" / "用女娲蒸馏"

## 蒸馏 vs 角色扮演

| 角色扮演 | 女娲蒸馏 |
|---------|---------|
| 模拟说什么 | 提取怎么想 |
| 一句看似有道理的话 | 3-7个心智模型同时审视 |
| 不可靠，没有认知框架支撑 | 结论有认知深度，可预测新问题立场 |

## 工作流程（四步蒸馏）

### Step 1: 六路并行采集
启动6个 subagent 同时调研：
1. **著作派** — 搜索此人的书籍、论文、公开写作
2. **播客派** — 搜索长对话、播客、访谈（权重最高）
3. **决策派** — 搜索实际商业/投资/管理决策记录
4. **社交派** — 搜索社交媒体发言、微博、推特
5. **批评派** — 搜索批评者视角（了解认知边界）
6. **时间线派** — 梳理人生关键节点和转折

信息源优先级：本人著作 > 长对话 > 实际决策 > 社交媒体 > 他人评价

### Step 2: 三重验证提炼
一个观点要被收录为"心智模型"，必须通过：
1. **跨域出现** — 在2+个不同领域/场景出现过（不是随口一说）
2. **有预测力** — 能推断对此人没讨论过的新问题的立场
3. **有排他性** — 不是所有聪明人都会这么想

三重全过 → **心智模型**
仅过1-2个 → **决策启发式**
零过 → **丢弃**

### Step 3: 构建 Skill
生成 SKILL.md，包含：
- **3-7个心智模型**（核心认知框架）
- **5-10条决策启发式**（实用判断规则）
- **表达DNA**（语言风格、修辞习惯、常用结构）
- **价值观与反模式**（坚信什么、绝不做什么）
- **诚实边界**（此人会承认不知道什么）

### Step 4: 质量验证
- 拿3个此人公开回答过的问题测试，方向一致才通过
- 用1个他没讨论过的问题测试，应表现出"适度不确定"而非"斩钉截铁"

## 产出格式

蒸馏完成后，Skill 文件写入 `skills/nuwa-distilled/{名字}/SKILL.md`

调用方式：
- "用芒格的视角帮我分析这个投资决策"
- "费曼会怎么解释量子计算？"
- "切换到 Naval 的视角"

## 蒸馏东东自己

当东东说"蒸馏我自己"时：
- 不做网络搜索
- 从 `memory/` 下的所有素材、日志、方案、决策记录中提取
- 从 SOUL.md、USER.md、MEMORY.md 中提取已知偏好
- 从 daily-capture 累积的素材中提炼思维模式
- 随着素材积累，东东的 Skill 会越来越精准
