# Dual-Track Planner / 双轨方案生成器

基于 OpenMythos RDT（循环深度Transformer）思维的双轨方案生成工具。

## 核心哲学

> **不是参数决定能力，而是循环深度决定质量。**

任何方案都有两个版本：
1. **传统版** - 单次深度推理，追求广度覆盖
2. **RDT版** - 循环迭代重构，追求深度穿透

## 使用场景

- 项目规划与里程碑设计
- 业务流程设计与优化
- 产品策略制定
- 运营方案设计
- 组织架构设计
- 任何需要系统性思考的方案类任务

## 执行流程

```
用户输入需求
    ↓
Step 1: [Prelude] 深度理解
    - 澄清用户真实意图
    - 识别约束条件
    - 确认成功标准
    ↓
Step 2: [Loop 1] 传统方案（深度推理）
    - 第一性原理拆解
    - 多元思维模型分析
    - 架构式编排
    - 输出完整方案A
    ↓
Step 3: [Loop 2] RDT重构（循环深化）
    - 识别方案A的核心假设
    - 用"循环深度替代参数规模"思维重构
    - 设计可迭代的循环机制
    - 输出RDT方案B
    ↓
Step 4: [Coda] 对比输出
    - 并排对比两版方案
    - 标注关键差异和取舍
    - 给出可执行建议
```

## 输出格式

### 第一部分：传统方案
- **核心理念**：一句话总结
- **关键假设**：列出3-5个关键假设
- **具体方案**：分阶段/分模块详述
- **资源配置**：人、财、物、时间
- **风险与应对**：主要风险点

### 第二部分：RDT方案
- **核心重构点**：与传统方案的本质差异
- **循环机制设计**：
  - Prelude（输入标准化）
  - Recurrent Block（循环核心）
  - Coda（输出沉淀）
- **深度指标**：如何衡量循环质量
- **自适应机制**：简单vs复杂场景的处理

### 第三部分：对比与建议
- **并排对比表**：关键维度对比
- **适用场景**：什么情况下选哪个
- **建议**：推荐方案 + 理由
- **下一步**：具体的第一个动作

## 思维框架

### RDT 核心原则

1. **用时间换空间**
   - 不是堆资源（人/钱/时间）
   - 而是用迭代深度换取质量

2. **权重共享**
   - 核心方法论/工具复用
   - 不重复造轮子

3. **三阶段架构**
   - Prelude：输入标准化
   - Recurrent Block：循环深化
   - Coda：输出沉淀

4. **隐式推理**
   - 能力在迭代中内化
   - 非一次性传授

5. **自适应计算**
   - 简单问题少循环
   - 复杂问题多循环

### 关键公式映射

```
OpenMythos: h_{t+1} = A·h_t + B·e + Transformer(h_t, e)

映射到方案设计：
第t+1轮交付 = 上一轮反馈 + 原始目标提醒 + 本轮深化
```

## 示例

### 示例1：Redpoint Phase 0 规划

**传统方案**：10人 + 3企，一次性完成

**RDT方案**：3人 × 3轮循环 = 9人次深度交付
- Loop 1：认知建设（理解任务要求）
- Loop 2：实战迭代（基于反馈优化）
- Loop 3：交付复盘（最终交付+成长总结）

**对比**：人数少70%，但每人能力成长3倍

### 示例2：内容生产系统

**传统方案**：每篇内容从头写，投入产出比低

**RDT方案**：
- Prelude：素材/观点/初稿（输入）
- Recurrent Block：
  - 第1轮：梳理核心论点
  - 第2轮：补充论据/案例
  - 第3轮：优化表达结构
  - 第4轮：检查逻辑漏洞
- Coda：最终成品（输出）

**对比**：同一素材多次迭代，质量显著提升

## 注意事项

1. **不要生搬硬套**
   - 不是所有方案都适合RDT
   - 简单明确的任务，传统方案更直接

2. **循环深度要合理**
   - 建议3-5轮为一个循环周期
   - 过多循环会导致"过思考"

3. **必须重新注入原始目标**
   - 每轮迭代都要回顾最初目标
   - 防止迭代过程中偏离方向

4. **关注收敛度**
   - 每轮改进幅度应该递减
   - 如果改进幅度增大，说明方向错了
