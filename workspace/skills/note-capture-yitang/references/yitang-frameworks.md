# 一堂课程常见框架与模型参考

## 一堂五步法

一堂最核心的创业方法论：

1. **需求**：验证需求是否存在、是否真实
2. **解决方案**：设计最小可行产品（MVP）
3. **商业模式**：如何赚钱、单位经济模型
4. **增长**：如何获取用户、规模化
5. **壁垒**：如何建立护城河、防御竞争

## 常见分析模型

### 需求验证
- 需求三角：痛点 × 频次 × 付费意愿
- 用户画像：人口属性 × 行为特征 × 心理特征
- 场景拆解：何时、何地、何人、何事、何因

### 商业模式
- 收入模型：一次性收费、订阅、抽成、广告
- 成本结构：固定成本 vs 变动成本
- 单位经济：CAC、LTV、毛利率、回本周期

### 增长策略
- AARRR模型：获客 → 激活 → 留存 → 收入 → 推荐
- 渠道矩阵：线上/线下、付费/免费、自建/合作
- 增长飞轮：核心动作 → 数据反馈 → 迭代优化

### 组织与团队
- 创业阶段：探索期 → 验证期 → 增长期 → 规模化期
- 核心团队：CEO（方向）× CTO（产品）× COO（运营）
- 股权设计：控制权、激励性、退出机制

## 一堂课程命名规律

常见课程名称格式：
- 【主题】+【方法】：如"需求验证：五步找到真需求"
- 【场景】+【解决方案】：如"冷启动：0预算获取前1000用户"
- 【数字】+【主题】：如"创业36条军规"

## 课件视觉特征

一堂课件通常包含：
- 标题页：课程名、讲师、所属模块
- 章节页：Part X + 章节名
- 内容页：要点 + 图示/案例
- 总结页：核心结论 + 行动建议
- 金句：加粗或高亮显示

## 笔记整理建议

1. **框架优先**：先提取模型/框架，再填充细节
2. **案例具体**：保留案例的关键数字和结果
3. **留白思考**："我的思考"部分建议课后回填
4. **行动导向**：每个课程至少提炼1个可执行动作
