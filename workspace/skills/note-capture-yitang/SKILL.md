# 一堂课件笔记捕获 Skill

## 核心功能

将一堂课程课件截图转换为结构化的学习笔记文档。

## 处理流程

### 场景A：yt + 图片同时发送
```
用户发送 "yt" + 课件截图
    ↓
1. 识别触发词"yt"
2. 提取消息中的图片
3. OCR提取文字内容
4. 分析课件结构
5. 生成结构化笔记
6. 创建飞书云文档
7. 返回文档链接
```

### 场景B：先发图片，后说yt
```
用户发送课件截图（无文字/无触发词）
    ↓
AI静默保存图片到上下文
    ↓
用户后续发送 "yt" 或 "yt 课程名"
    ↓
1. 识别触发词"yt"
2. 从上下文中提取最近一张图片
3. OCR提取文字内容
4. 分析课件结构
5. 生成结构化笔记
6. 创建飞书云文档
7. 返回文档链接
```

**上下文关联规则**：
- 查找当前对话中最近的图片（5分钟内的有效）
- 如果找不到图片，询问东东"请发送一堂课件截图"
- 如果找到多张图片，询问"检测到多张图片，请确认处理哪一张"

## 执行步骤

### Step 1: 接收并保存图片

用户使用 `/note` 命令发送课件截图后：
1. 保存图片到 `/workspace/projects/workspace/media/notes/YYYYMMDD-HHMMSS.jpg`
2. 使用 `image` 工具分析图片内容

### Step 2: 提取并分析内容

使用 `image` 工具时，prompt 要求：
```
请详细提取这张一堂课程课件截图中的所有文字内容，并按以下结构分析：

1. 课程主题和副标题
2. 核心框架/模型（如有图示，请描述）
3. 关键概念定义
4. 案例（背景、动作、结果）
5. 方法论/工具
6. 金句或重要结论

请保持原文的层次结构，标注清楚章节关系。
```

### Step 3: 生成结构化笔记

基于 image 工具返回的内容，填充到「一堂笔记架构模板」中：
- **课程概览**：提取主题、适用场景、结构
- **核心框架**：提取模型名称、定义、要素
- **关键概念**：整理概念表格
- **案例与实证**：提取案例详情
- **方法论与工具**：整理操作步骤
- **我的思考**：留白（标记为【待填写】）
- **行动建议**：留白（标记为【待填写】）
- **原文摘录**：提取金句

### Step 4: 创建飞书云文档

使用 `feishu_create_doc` 工具创建文档：
- 标题格式：`【课程名】学习笔记 — YYYY-MM-DD`
- 内容：填充好的结构化笔记
- 保存位置：东东的云空间（默认）

### Step 5: 返回结果

向东东返回：
```
✅ 课件笔记已生成

📄 飞书文档：[链接]

📊 内容摘要：
- 课程主题：[主题]
- 核心框架：[框架数量] 个
- 关键概念：[概念数量] 个
- 案例：[案例数量] 个
- 待填写部分：我的思考、行动建议

💡 建议：观看课程视频后，回填"我的思考"部分，形成完整学习闭环。
```

## 使用方法

### 方式1：yt + 图片（同时发送）
```
yt

```

### 方式2：yt + 课程名 + 图片
```

```

### 方式3：先发图片，后说yt（自动关联）
```
[贴上一堂课件截图]  ← 先发图

yt                    ← 后说yt，自动关联上图
```

### 方式4：关键词触发
```
一堂课件笔记
[贴上一堂课件截图]
```

## 特殊情况处理

### 图片质量不佳
如果 OCR 提取效果差：
- 告知东东"图片文字识别效果有限，建议："
  1. 重新发送更清晰的截图
  2. 或手动提供课程名称，我创建空白模板

### 多页课件
如果东东发送多张截图：
- 分别处理每张图片
- 合并到同一个文档中，按页码组织

### 非一堂课件
如果识别出不是一堂课件：
- 仍按相同流程处理
- 但在文档头部标注"非一堂来源"，方便后续分类
