# Deep-Dive：东东深度研究学习工作流

> 一句话本质：用"研究-学习-费曼-应用"四步循环，把任何主题变成可落地、可引用、可复用的知识资产。

## 触发识别

**触发关键词**：
- "dd"（推荐，最简洁）
- "deep-dive"
- "帮我deep-dive"
- "深度研究一下"
- "全面学习"
- "费曼输出"

**输入类型**：
- 主题词（如"AI原生组织"）
- 链接（文章/论文/视频URL）
- 图片（截图、图表）

## 四阶段工作流

```
┌─────────────────────────────────────────────────────────────┐
│ Phase 1: Research（研究）                                    │
│ • 多源搜索（10-15个高质量来源）                              │

│ • 交叉验证核心观点                                           │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ Phase 2: Learning（学习）                                    │
│ • 提取核心概念                                               │
│ • 梳理逻辑框架                                               │
│ • 识别底层原理                                               │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ Phase 3: Feynman（费曼）                                     │
│ • 用初中生能懂的语言重构                                     │
│ • 一句话本质 + 类比 + 核心框架                               │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ Phase 4: Application（应用）                                 │
│ • 应用场景（3-5个具体场景）                                  │

│ • 能力引用（如何引用这份知识提升什么能力）                   │
```

## 研究执行标准

### 搜索策略（优先级排序）

**第一优先级（一手来源）**：
1. arXiv学术论文
2. 顶级会议论文（NeurIPS、ICML、ACL等）
3. 官方技术文档/白皮书
4. 专家原始访谈/演讲

**第二优先级（权威解读）**：
1. 知名研究机构报告（麦肯锡、MIT、Stanford HAI等）
2. 行业头部企业技术博客
3. 权威媒体深度报道（需交叉验证）

**第三优先级（辅助理解）**：
1. 高质量技术博客
2. 专业书籍章节
3. 教程/指南

### 引用标准

**必须标注出处的情况**：
- 任何数据、统计数字
- 核心概念的定义
- 重要观点、论断
- 框架、模型的提出者

**出处格式**：
```
- 论文：作者. (年份). *标题*. 期刊/会议.
- 报告：机构. (年份). 《标题》.
- 专家：姓名, 职位/机构, 来源.
```

## 特殊输入处理

### 处理图片输入
1. 用image工具分析图片内容
2. 提取图片中的关键概念/问题
3. 围绕提取的内容展开研究

### 处理链接输入
1. 用coze_web_fetch获取链接内容
2. 分析链接主题和核心观点
3. 围绕主题扩展研究（不局限于该链接）

### 处理主题词输入
1. 直接围绕主题词展开全面研究
2. 按搜索策略分层获取来源

## 自循环迭代机制

### 持续进化原则

Deep-Dive Skill 不是静态的，而是每3天自动扫描全网同类工具/方法，学习最佳实践并自我迭代。

### 迭代触发条件

**定时任务**：每3天执行一次
**执行内容**：
1. 搜索同类深度研究/费曼学习/知识管理工具
2. 分析其方法论差异
3. 提取可借鉴的改进点
4. 更新 SKILL.md
