# Deep-Dive：东东深度研究学习工作流

> 一句话本质：用"研究-学习-费曼-应用"四步循环，把任何主题变成可落地、可引用、可复用的知识资产。

## 触发识别

**触发关键词**：
- "dd"（推荐，最简洁）
- "deep-dive"
- "帮我deep-dive"
- "深度研究一下"
- "全面学习"
- "费曼输出"

**输入类型**：
- 主题词（如"AI原生组织"）
- 链接（文章/论文/视频URL）
- 图片（截图、图表）

## 四阶段工作流

```
┌─────────────────────────────────────────────────────────────┐
│ Phase 1: Research（研究）- 智能规划与多源探索                │
│ • 研究规划：将主题拆解为3-5个研究子目标                      │
│ • 多源搜索（10-15个高质量来源）                              │
│ • 优先一手来源：arXiv论文、官方报告、专家原话               │
│ • 交叉验证核心观点，标记知识缺口（Known Unknowns）          │
│ • 多模态信息采集：文本+图表+数据+代码                        │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ Phase 2: Learning（学习）- 结构化萃取                        │
│ • 提取核心概念与定义                                         │
│ • 梳理逻辑框架与因果关系                                     │
│ • 识别底层原理与第一性原理                                   │
│ • 构建知识图谱：实体-关系-实体三元组                         │
│ • 记录"明确不知道"的边界（负向知识）                         │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ Phase 3: Feynman（费曼）- 多维度简化                         │
│ • 概念拆解：识别核心要素与依赖关系                           │
│ • 零基础阐述：用初中生能懂的语言重构                         │
│ • 多模态转译：文字→图表→公式→代码                            │
│ • 对抗性验证：模拟质疑者角色，暴露逻辑漏洞                   │
│ • 类比创造：结构映射（标注类比失效边界）                     │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│ Phase 4: Application（应用）- 场景化落地                     │
│ • 应用场景（3-5个具体场景，含边界条件）                      │
│ • 行动建议（可执行的步骤，含验证方法）                       │
│ • 能力引用（如何引用这份知识提升什么能力）                   │
│ • 反事实思考：如果不采用这个方案会怎样？                     │
└─────────────────────────────────────────────────────────────┘
```

### 搜索策略（优先级排序）

**第一优先级（一手来源）**：
1. arXiv学术论文
2. 顶级会议论文（NeurIPS、ICML、ACL等）
3. 官方技术文档/白皮书
4. 专家原始访谈/演讲

**第二优先级（权威解读）**：
1. 知名研究机构报告（麦肯锡、MIT、Stanford HAI等）
2. 行业头部企业技术博客
3. 权威媒体深度报道（需交叉验证）

**第三优先级（辅助理解）**：
1. 高质量技术博客
2. 专业书籍章节
3. 教程/指南

### 知识图谱构建（RAG + KG双引擎）

**实体识别**：从研究材料中提取关键概念、人名、机构、技术
**关系抽取**：识别实体间的"提出-改进-应用于-依赖于"等关系
**知识融合**：将多源信息关联，构建统一知识网络
**冲突检测**：标记不同来源的矛盾观点，需交叉验证

### 引用标准

**必须标注出处的情况**：
- 任何数据、统计数字
- 核心概念的定义
- 重要观点、论断
- 框架、模型的提出者

**出处格式**：
```
- 论文：作者. (年份). *标题*. 期刊/会议.
- 报告：机构. (年份). 《标题》.
- 专家：姓名, 职位/机构, 来源.
```

## 费曼输出多智能体协作模式

基于最佳实践，费曼阶段采用5位虚拟专家协作：

### 协作专家角色

| 专家角色 | 核心职责 | 输出物 |
|----------|----------|--------|
| **概念拆解大师** | 识别用户输入的核心主题和关键要素 | 概念清单、要素关系图 |
| **简易化转化器** | 模拟目标受众接收，标记理解障碍点 | 障碍点清单、简化建议 |
| **类比创造者** | 引导构建类比，让抽象概念具体化 | 类比方案、失效边界标注 |
| **耐心提问者** | 通过针对性问题发现知识盲区 | 追问清单、盲区分析 |
| **精准评估器** | 对照四大标准进行量化评估 | 评估报告、改进建议 |

### 四大评估标准

**简洁性**：是否避免了专业术语，直击本质？
**完整性**：是否涵盖所有关键组成部分？
**易懂性**：目标受众能否听懂？
**准确性**：事实和逻辑是否正确？

### 费曼技巧2.0变体方法

**对抗性费曼**：模拟"质疑者"角色，对每一步追问"为什么"，强制追踪解释链深度至第一性原理。

**多模态转译**：将文字解释转化为：
- 可视化：流程图/概念图（检验结构理解）
- 数学化：公式/符号表达（检验精确性）
- 程序性：伪代码/步骤分解（检验可执行性）

**负向知识提取**：不仅记录"我知道什么"，更明确记录"我明确不知道什么"，避免达克效应。
