# Daily Capture — 素材驱动的每日内容生产

## 核心逻辑

<callout emoji="⚡" background-color="light-blue">
**"n" 前缀 = 素材模式**。收到 `n` 开头的消息时：静默归档（NO_REPLY），启动后台 subagent 深化研究。
**无前缀 = 正常对话**。正常回应。
</callout>

## 消息处理规则

### 当东东发的消息以 `n` 开头（或消息内容明显是素材碎片）

**立即执行：**
1. 回复 `NO_REPLY`（不打扰东东的思路）
2. 识别素材类型：
   - 文字 → 想法/洞察/记录
   - 图片 → 视觉记录（用 `image` 工具分析内容）
   - 文件/语音 → 保存并转写（用 `coze-asr`）
   - 截图 → 信息记录
3. 归档到当日素材文件：`memory/daily-raw/YYYY-MM-DD.md`
   - 格式：`[HH:MM] [类型] 素材内容`

**后台启动 subagent（异步，不影响对话）：**
- 启动一个 subagent 研究深化这条素材
- subagent 任务：
  1. 识别素材关联的领域（产业/AI/创业/个人成长等）
  2. 搜索相关背景信息（用 coze_web_search）
  3. 查找类似案例/数据/引用
  4. 基于对东东的理解，提出3-5个延伸思考方向
  5. 将研究结果写入 `memory/daily-research/YYYY-MM-DD-{序号}.md`
- subagent 完成后自动 announce，不主动推送（等晚间汇总）

### 判断是否为素材模式

以下情况视为素材，即使没有 `n` 前缀：
- 东东发了一张/多张照片
- 东东发了一段语音/视频
- 东东发了一个文件
- 东东说"记录一下"、"素材"、"这个可以写"等
- 东东转发了别人的消息并附加简短评论

以下情况**不是**素材，正常回应：
- 东东在问问题
- 东东在给指令
- 东东在跟人聊天
- 东东明确说"回复我"、"你觉得呢"

<callout emoji="🗺️" background-color="light-yellow">
**每条素材都必须关联至少一个板块。板块之间互相支撑，最终汇聚到纪东不已IP。**
</callout>

| 板块 | 定位 | 内容类型 | 素材标签 |
|------|------|---------|---------|
| **一堂云南学习中心** | 方法论落地场域 | 沙龙、案例拆解、学员故事、方法论实践 | `#学习中心` |
| **德鲁克精读社** | 深度内容+读书社群 | 读书笔记、管理洞见、经典拆解、读书会 | `#德鲁克` |
| **OPC/硅创联昆明中心** | 产业对接+AI赋能 | 产业对接、AI工具实操、会员案例、技术赋能 | `#OPC` |
| **纪东不已投资** | 实操能力展示 | 投资逻辑、项目分析、商业判断、产业洞察 | `#投资` |
| **大航海项目** | 当前重点项目 | 白芸豆、玲素品牌、商业突破进展 | `#大航海` |

**板块联动规则：**
- 学习中心的活动素材 → 可延伸为公众号文章（产业认知支柱）
- 德鲁克精读 → 可延伸为公众号文章（读书心得支柱）
- OPC产业对接 → 可延伸为投资分析（产业认知支柱）
- 大航海进展 → 所有板块的内容源
- **一篇好文章 = 素材来自板块A + 洞察来自板块B + 认知来自板块C**

```markdown

## 素材清单

### [HH:MM] 类型：文字/想法 | 板块：学习中心

### [HH:MM] 类型：照片 | 板块：大航海
```

## 后台研究 Subagent 模板

subagent 的 task prompt：

```
你是一个研究助手。东东今天发来了一条素材，请深度研究并延展。

## 素材内容
{素材内容}

## 东东的背景（用于理解素材上下文）
- 身份：产业架构师，健康行业（白芸豆/玲素品牌）
- 当前项目：白芸豆大航海、一堂云南学习中心、纪东不已IP
- 内容支柱：产业认知、AI实践、读书心得、灵感反思
- 风格：第一性原理、多元思维、深度但不学术

## 研究任务
1. 识别素材关联的领域和话题
2. 搜索相关背景信息（行业数据、案例、趋势）
3. 找到3-5个可延伸的方向
4. 思考：基于对东东的理解，这些素材可以连接到哪些项目/方案
5. 提出2-3个东东"可能没想到"的角度

## 输出
写入 {research_file_path}
格式：
### 素材分析
### 背景研究
### 延伸方向
### 连接建议（关联到哪些项目/方案）
### 未解答的问题（东东可以继续深挖的点）
```

## 晚间 21:00 流程（cron 触发）

### Step 1: 素材盘点

读取 `memory/daily-raw/YYYY-MM-DD.md`，统计今日素材数量和类型。

### Step 2: 日志生成（尽量还原）

基于素材 + 研究结果，生成日志：

```
📝 Day N · 素材日志（M月D日）

📋 今日素材（X条）
- [HH:MM] 类型 — 一句话描述
- ...

🔄 今日还原
用叙事体还原东东今天的一天。不是罗列，是讲故事。
包括：去了哪里、看到了什么、想到了什么、跟谁聊了什么。

🔍 深化研究发现
- 素材A关联：[研究结论摘要]
- 素材B关联：[研究结论摘要]
- ...

💡 延伸方向（我认为值得深挖的）
1. [方向一] — 理由
2. [方向二] — 理由

❓ 我还想了解的（需要东东补充的）
- [问题一]
- [问题二]

✍️ 推荐写作方向
A. [方向] — 为什么值得写
B. [方向] — 为什么值得写

请确认或调整 👇
```

### Step 3: 推送确认

将日志发给东东，等待确认。

### Step 4: 写作

东东确认核心点后：
1. 启动 Writer subagent（传入素材+研究方向+风格指南）
2. Writer 完成后启动 Prism subagent 质量审查
3. 审查通过后推送终稿
4. 东东终审发布

### Step 5: 归档

- capture 到 memory-cli.sh（event 类型）
- 更新 `memory/content-topic-library.md`（标记已用选题）
- 清理当日研究文件（移入 archive）

```
## 4. 内容资产评估
```

| `feishu-bitable` | 内容效果追踪表 |

## 注意事项

- **不要主动追问**：东东发什么收什么，不问"还有吗"
- **不要在素材模式下展开长对话**：NO_REPLY 是默认行为
- **研究 subagent 要轻量**：每个素材的研究控制在 2-3 分钟内，不要深度展开
- **日志要像故事**：不是罗列清单，是还原一天的经历
- **深化报告要克制**：观察和建议要基于事实，不要臆测东东的内心
